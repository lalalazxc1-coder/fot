# 🚀 План на завтра (Финальные штрихи и Продакшен)
> Этот список содержит задачи для завершения проекта, направленные на стабильность, архитектуру и подготовку системы к выкату в production. Опирается на то, что первые два этапа (Security Audit v1 и v2) успешно пройдены.

---

## 🏗️ 1. Архитектура и База Данных (Backend)

- [x] **Внедрение Alembic для миграций БД**
  - *Зачем:* Сейчас в `main.py` используется ⚠️ `Base.metadata.create_all()`. Это неприемлемо для продакшена, так как не позволяет менять структуру таблиц без потери данных.
  - *Задача:* Настроить `alembic init`, сгенерировать начальную миграцию и перевести проект на версионирование схемы БД.

- [x] **Redis Token Blacklist (Для честного Logout)**
  - *Зачем:* JWT по своей природе stateless. Даже если мы удалили куку на клиенте, украденный токен все еще валиден до истечения срока (`exp`).
  - *Задача:* Поднять Redis, записывать ID токена (или сам токен) в черный список при логауте с TTL, равным оставшемуся времени жизни токена. В мидлвари проверять токен по Redis.

- [x] **Refresh Token механика**
  - *Зачем:* Сейчас `access_token` выдаётся на очень долгий срок (или на месяц при "Remember Me"). Это небезопасно.
  - *Задача:* Сделать короткоживущий `access_token` (например, 15 минут) и долгоживущий `refresh_token` (в HttpOnly куке). Написать эндпоинт `/api/auth/refresh`.

## 🎨 2. Доработка Фронтенда (TypeScript & React)

- [x] **Полное искоренение `any` (Финальная чистка)**
  - *Зачем:* У нас остались заглушки `any` в самых тяжелых и сложных файлах (около 60 штук), таких как `MarketPage.tsx`, `JobOffersPage.tsx`, `ScenariosPage.tsx`. Из-за этого мы теряем мощь автодополнения и рискуем упасть в рантайме.
  - *Задача:* Объявить строгие интерфейсы для сырых данных `entries`, `groupedData` и деревьев DOM в React.

- [x] **Глобальный обработчик ошибок (Error Boundary / Interceptor)**
  - *Зачем:* Сделать красивые попапы / редиректы при падении бэкенда.
  - *Задача:* Если Axios получает ошибку 401/403 (например, истек `refresh_token` или кука удалена), автоматически разлогинить юзера, очистить стейт и показать Toaster ("Сессия истекла").

## 🧪 3. Тестирование (QA)

- [ ] **Интеграционные тесты для Backend (Pytest)**
  - *Зачем:* Убедиться, что рефакторинг безопасности не сломал бизнес-логику для реальных юзеров.
  - *Задача:* Подключить `pytest`, настроить in-memory SQLite (или тестовую БД Postgres) и написать тесты на основные цепочки: создание плана -> расчет ФОТ -> аналитика.

- [ ] **Frontend тесты (Vitest / RTL)**
  - *Зачем:* Гарантировать, что UI рендерит корректные метрики.
  - *Задача:* Тесты для сложных компонентов (например, калькуляторов зарплат и сценариев).

## 🚀 4. Инфраструктура и Деплой (DevOps)

- [x] **Dockerization**
  - *Зачем:* Чтобы любой разработчик (и сервер) мог развернуть приложение одной кнопкой.
  - *Задача:* Написать `Dockerfile` для FastAPI (бэкенд) и `Dockerfile` для Vite (в идеале multi-stage build c Nginx для раздачи статики). Обернуть всё в `docker-compose.yml` вместе с PostgreSQL и Redis.

- [ ] **Валидация всех Pydantic & Response моделей**
  - *Зачем:* Избавление от протечек внутренней структуры БД (например, скрытого поля hashed_password, хотя оно уже закрыто, нужно проверить остальные модели).
  - *Задача:* Стандартизировать `response_model` во всех `APIRouter()`. Форсировать единообразную структуру выдачи ошибок (например, всегда возвращать `{"status": "error", "message": "..."}`).
