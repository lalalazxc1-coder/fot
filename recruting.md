# Статус разработки системы рекрутинга

## Актуальный итог

Основной объем задачи по рекрутингу закрыт: бэкенд расширен, формы и UI на фронтенде доведены, локальный чат по кандидату работает, загрузка резюме подключена.

## ✅ Что уже сделано

### Бэкенд (API и БД)

- Расширены модели и схемы для рекрутинга:
  - `Vacancy`: `position_name`, `description`, `salary_from`, `salary_to`.
  - `Candidate`: `phone`, `email`, `resume_url`.
- Реализован эндпоинт загрузки резюме:
  - `POST /candidates/{id}/resume` (сохранение в `/uploads/resumes/`).
- Улучшены уведомления заказчику:
  - Сообщение формируется от имени реального отправителя (Full Name).
  - Ссылка ведет сразу в нужную заявку и вкладку воронки.

### Фронтенд (UI/UX и функционал)

- Доработана форма создания заявки в `frontend/src/pages/JobRequestsPage.tsx`:
  - Должность выбирается из справочника (`usePositions`).
  - Добавлено обязательное поле "Что должен делать кандидат".
  - Добавлены поля зарплатной вилки `от/до` с валидацией диапазона.
- Доработана форма добавления кандидата в `frontend/src/pages/RecruitingPage.tsx`:
  - Добавлены поля `Телефон` и `Email` с валидацией.
  - Добавлена кнопка/поле "Прикрепить резюме" (PDF/DOC/DOCX).
  - После создания кандидата резюме отправляется на `/candidates/{id}/resume`.
- В деталях вакансии для рекрутера отображается зарплатная вилка (если заполнена).
- Локальный чат по кандидату уже работает через существующий `CommentsSection`:
  - При выборе кандидата комментарии привязываются к `targetType='candidate'`.
  - Без выбранного кандидата показывается общий чат вакансии (`targetType='vacancy'`).
- Compact UI и локализация сохранены:
  - Компактный канбан и карточки.
  - Русские названия этапов.
  - Исправленный скролл и автоскролл комментариев.
