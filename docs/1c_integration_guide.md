# Документация интеграции: Сайт ФоТ <-> 1С:ЗУП 3.1

Эта документация содержит описание реализованного механизма интеграции, настройки на стороне 1С и инструкции для дальнейшей разработки.

---

## 1. Архитектура интеграции

Интеграция построена на базе **HTTP-сервисов 1С**.
- **Сайт (Backend)** выступает в роли клиента.
- **1С:ЗУП** выступает в роли сервера.
- Данные передаются в формате **JSON**.
- Авторизация: **Basic Auth** (логин/пароль пользователя 1С).

---

## 2. Настройка на стороне 1С

### Объект в конфигурации
1. **HTTP-сервис**: `HTTPСервис1`
   - **Корневой URL**: `mybase`
2. **Шаблон URL**: `GetData`
   - **Шаблон**: `/get_fio`
3. **Метод**: `GET`
   - **Обработчик**: `СписокСотрудниковGet`

### Код модуля (1С)
```1c
Функция СписокСотрудниковGet(Запрос)
    УстановитьПривилегированныйРежим(Истина);
    
    СписокФИО = Новый Массив;
    Выборка = Справочники.Сотрудники.Выбрать();
    
    Счетчик = 0;
    Пока Выборка.Следующий() И Счетчик < 200 Цикл 
        Если Выборка.ПометкаУдаления Тогда Продолжить; КонецЕсли;
        СписокФИО.Добавить(Строка(Выборка.Наименование));
        Счетчик = Счетчик + 1;
    КонецЦикла;
    
    ЗаписьJSON = Новый ЗаписьJSON;
    ЗаписьJSON.УстановитьСтроку();
    ЗаписатьJSON(ЗаписьJSON, СписокФИО);
    ОтветТекст = ЗаписьJSON.Закрыть();
    
    Ответ = Новый HTTPСервисОтвет(200);
    Ответ.УстановитьТелоИзСтроки(ОтветТекст, КодировкаТекста.UTF8);
    Возврат Ответ;
КонецФункции
```

### Права доступа (Роль)
Создана роль `https`:
- **HTTPСервис1**: право "Использование".
- **Справочник.Сотрудники**: права "Чтение" и "Просмотр".
- Роль назначена пользователю, под которым идет подключение (напр. `admin`).

## 3. Настройка на стороне Сайта

### Состав компонентов
1. **Backend Service** (`services/onec_service.py`):
   - Класс `OneCService`: отвечает за выполнение HTTP-запросов и обработку ошибок.
   - Метод `test_connection`: проверка связи.
   - Метод `get_employees`: получение списка ФИО.
2. **Backend Router** (`routers/integrations/onec.py`):
   - `GET /api/integrations/onec/employees`: прокси-метод для фронтенда.
   - `POST /api/integrations/onec/import`: массовое создание сотрудников в базе сайта. Защищает от дублей по ФИО и пишет лог в Audit Log.
3. **Frontend components**:
   - `IntegrationsPage.tsx`: Настройка URL, логина и пароля. Пароль шифруется перед сохранением.
   - `ImportOneCModal.tsx`: Интерфейс выбора сотрудников для импорта из 1С.
   - `EmployeeTable.tsx`: Кнопка "Импорт из 1С" (отображается только при активной интеграции).

### Важные нюансы
- **URL для Docker**: При работе сайта в Docker-контейнере, а 1С на хост-машине (Windows), необходимо использовать адрес `http://host.docker.internal/[имя_базы]`.
- **Пути**: Бэкенд автоматически достраивает путь `/hs/mybase/get_fio` к базовому URL.
