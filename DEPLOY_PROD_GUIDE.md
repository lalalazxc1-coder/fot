# Краткая инструкция: Развертывание проекта (Docker + Production)

Эта инструкция поможет вам запустить проект на боевом (Linux) сервере с помощью Docker точно так же, как вы это сделали локально.

## Шаг 1. Подготовка сервера

1. Установите на сервере **Docker** и **Docker Compose**.
2. Скопируйте папку с проектом (`fot`) на ваш сервер (например, при помощи `git clone` или просто скопировав файлы через `scp`).

---

## Шаг 2. Настройка файла `.env`

В корне проекта (рядом с `docker-compose.yml`) создайте или откройте файл `.env`. 
Настройте в нём безопасные данные для продакшена:

```env
# 1. Задайте сложный секретный ключ (важно для безопасности сессий!)
SECRET_KEY=ВАШ_СЛОЖНЫЙ_ДЛИННЫЙ_КЛЮЧ

# 2. Укажите логин и пароль для первого входа (чтобы создать админа)
ADMIN_EMAIL=ваша@реальная.почта
ADMIN_PASSWORD=супер_сложный_пароль

# 3. URL-адреса, с которых разрешен доступ к вашему серверу (CORS)
# Если у вас сайт будет на домене "example.com", укажите его:
ALLOWED_ORIGINS=https://example.com,http://example.com

# 4. Включите боевой режим:
ENVIRONMENT=production
LOG_LEVEL=INFO
```

---

## Шаг 3. Подготовка docker-compose.yml к продакшену (Важно!)

Для безопасности, лучше закрыть внешний доступ к базе данных и Redis (чтобы хакеры не пытались подобрать пароль).
Для этого откройте `docker-compose.yml` и удалите или закомментируйте секцию `ports` у сервисов `db` и `redis`:

```yaml
  db:
    image: postgres:15-alpine
    container_name: fot_db
    # ports:                    <-- Закомментируйте эти две строки
    #   - "5433:5432"           <--
    ...
    
  redis:
    image: redis:7-alpine
    container_name: fot_redis
    # ports:                    <-- И вот эти
    #   - "6379:6379"           <--
```
*Бэкенд всё равно сможет общаться с ними внутри виртуальной сети Docker.*

---

## Шаг 4. Сборка и запуск контейнеров

Перейдите в папку с проектом (`cd /путь/к/проекту/fot`) и выполните команду:

```bash
docker-compose up --build -d
```
*Что это делает: Скачивает образы, собирает фронтенд и бэкенд, и запускает все 4 контейнера в фоновом режиме (`-d`).*

---

## Шаг 5. Применение миграций и создание первого Админа

Таблиц в чистой базе пока нет. Нам нужно их создать:

```bash
docker exec -t fot_backend bash -c "alembic upgrade head && python init_db.py"
```
*Что это делает: Подключается к работающему контейнеру бэкенда (`fot_backend`), применяет все настройки таблиц (миграции) и запускает скрипт, который создаст пользователя Админа из файла `.env`.*

**✅ Готово! Ваш проект запущен и готов к работе.** 
* Фронтенд работает на 80 порту (будет доступен по IP сервера или домену).
* Бэкенд (и его документация `/docs`) - на 8000 порту.
